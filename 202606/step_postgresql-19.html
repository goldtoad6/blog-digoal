Product Capabilities

PostgreSQL 19 is a milestone release with 212 updates spanning graph queries, DML improvements, query planning, maintenance operations, operational tooling, and upgrade compatibility. Here are the key capabilities that make it the most operationally mature release yet.

🔗
Native Graph Queries
Implements the SQL/PGQ standard (ISO SQL:2023 Part 16), enabling property graph queries on existing relational tables. Use Cypher-style pattern matching for traversals—no separate graph database needed. Internally rewritten as relational operations using your existing indexes.
SQL:2023
⚡
Atomic Get-Or-Create
Adds INSERT ... ON CONFLICT DO SELECT ... RETURNING, completing the upsert trilogy. Return conflicting rows atomically, optionally locking them with FOR UPDATE/FOR SHARE—eliminating the need for read-then-insert race conditions.
DML
♻️
Online Table Repack
A new REPACK command replaces VACUUM FULL and CLUSTER. With CONCURRENTLY, rebuild tables without access-exclusive locking—eliminating hours of downtime for bloated tables in production environments.
Maintenance
🔄
Parallel Autovacuum
Autovacuum can now use parallel workers via autovacuum_max_parallel_workers and per-table autovacuum_parallel_workers. Large tables are cleaned up significantly faster by parallelizing index and heap vacuuming.
Performance
📊
Adaptive Async I/O
Async I/O moves from static io_workers to a dynamic Worker Pool. New parameters (io_min_workers, io_max_workers, io_worker_idle_timeout) auto-scale based on load—making async I/O production-ready with default settings.
I/O Subsystem
🎯
Query Plan Hints
The new pg_plan_advice module lets you stabilize and influence optimizer decisions. Combined with pg_stash_advice for per-query-id advice, you can fine-tune execution plans without modifying SQL—a long-awaited DBA tool.
Planner
