Core Highlights

六大值得立即关注的特性

从图查询到自动调优，PostgreSQL 19 在性能、开发者体验与可观测性上同时跃迁。

NEW

SQL/PGQ 图形查询

原生支持 SQL Property Graph Queries 标准，像写视图一样编写关系型图查询，无需安装额外扩展。

NEW

UPDATE/DELETE FOR PORTION OF

时态数据操作原生支持，针对时间范围执行行级变更。无需触发器，时序场景开发效率倍增。

NEW

REPACK 取代 VACUUM FULL

统一替代 VACUUM FULL 与 CLUSTER，支持 CONCURRENTLY 无锁重建，业务不中断。

PERF

SIMD 全方位加速

COPY、CRC32C、页面校验和、hex 编解码全面启用 AVX2 / ARM Crypto 指令，吞吐显著提升。

PERF

LZ4 默认压缩

TOAST 压缩默认从 pglz 切换到 LZ4，压缩/解压更快更省 CPU，大字段存储普遍受益。

OPS

并行 Autovacuum

autovacuum 支持并行 worker，引入评分系统决定表清理顺序，运维更智能、回收更及时。

SQL Capabilities

全新 SQL 语法与开发者体验

更少的样板代码、更直接的表达力，让日常开发与运维脚本更加简洁。

INSERT … ON CONFLICT DO SELECT

冲突时可返回目标行并加锁，配合 RETURNING 一气呵成处理 upsert 中的复杂业务。

  • 原生时态操作UPDATE/DELETE FOR PORTION OF 直接在 SQL 中表达时间窗逻辑
  • 更少的胶水代码无需触发器或 CTE 模拟，标准 SQL 一行搞定
  • 更强一致性RETURNING 与 FOR UPDATE/SHARE 协同，避免竞态读取
-- 时态表：仅更新 2026 年 6 月的数据
UPDATE orders
   SET status = 'archived'
 WHERE id BETWEEN 1000 AND 5000
   FOR PORTION OF valid_during
       FROM '2026-06-01' TO '2026-07-01';

-- 冲突时返回已存在行并锁定
INSERT INTO accounts(name, balance)
VALUES ('alice', 100)
ON CONFLICT (name) DO SELECT
   RETURNING * FOR UPDATE;

更友好的 GROUP BY · 窗口函数 · COPY

GROUP BY ALL 自动按非聚合列分组，窗口函数支持 IGNORE NULLS，COPY 支持 JSON 输出。

  • GROUP BY ALL告别重复列名列表，写 SQL 像写自然语言
  • lead/lag/nth_value支持 IGNORE/RESPECT NULLS，分析函数更强大
  • COPY TO JSON直接输出 JSON 数组，配合 FORCE_ARRAY 一行即 API
-- 自动按所有非聚合列分组
SELECT region, product, SUM(amount) AS total
FROM sales
GROUP BY ALL;

-- 窗口函数跳过 NULL
SELECT
  lag(amount) IGNORE NULLS OVER (ORDER BY ts) AS prev_amt,
  amount - lag(amount) IGNORE NULLS OVER (ORDER BY ts) AS delta
FROM cashflow;

-- COPY 直接输出 JSON 数组
COPY (SELECT * FROM products) TO '/tmp/p.json'
  (FORMAT JSON, FORCE_ARRAY);

异步 I/O 改进

大请求预读调度优化，io_method=worker 自动调整后台 worker 数量，磁盘 I/O 更高效。

TID 范围扫描并行化

按 TID 范围访问堆表时自动启用并行执行，单表全表扫描更快速。

基数排序

排序算法升级到 Radix Sort，大数据量 ORDER BY 性能显著提升。

纳秒级计时

EXPLAIN ANALYZE 与 pg_test_timing 默认纳秒精度，更精确的性能分析。

哈希索引批量删除

使用流式读取改进哈希索引 bulk delete 与 GIN 索引 clean 性能。

优化器多项增强

NOT IN 转反连接、更多 LEFT JOIN 转 ANTI JOIN、连接选择性计算加速。

Observability

看得更清楚、调得更精准

PostgreSQL 19 引入大量新视图、列与等待事件，DBA 终于能"看见"以前看不见的状态。

pg_stat_lock

按锁类型分组的统计视图，快速定位热点锁竞争与阻塞链路。

pg_stat_recovery

恢复状态实时可视化，备机追赶进度一目了然。

pg_stat_autovacuum_scores

每表 autovacuum 评分详情，结合新权重系统让调度更可解释。

EXPLAIN (ANALYZE, IO)

新增 IO 报告选项，异步 I/O 活动首次在执行计划中可见。

pg_get_*_ddl()

新增角色、表空间、数据库的 DDL 输出函数，备份/迁移脚本更易编写。

process-type 日志粒度

log_min_messages 支持按进程类型独立设置，审计/调试更精细。

Compatibility & Migration

升级前需要知道的事

PostgreSQL 19 包含少量破坏性变更，请提前评估影响范围并使用 pg_upgrade 或逻辑复制平滑迁移。

REMOVED
移除 RADIUS 认证
基于 UDP 的 RADIUS 存在不可修复的安全问题，请改用其它认证方式。
高影响
CHANGED
inet/cidr 默认索引 opclass
由 btree_gist 切换到 GiST，pg_upgrade 检测到旧索引将拒绝升级。
高影响
CHANGED
max_locks_per_transaction 默认 64 → 128
锁大小分配已变更，要匹配之前容量需将默认值翻倍。
中影响
CHANGED
standard_conforming_strings 强制为 on
escape_string_warning 服务器变量被移除；旧客户端仍可关闭该选项以兼容。
中影响
CHANGED
JIT 默认禁用
优化器成本计算不再可靠地启用 JIT，可手动开启或基于 workload 调优。
中影响
NEW
密码过期警告 & MD5 警告
password_expiration_warning_threshold 与 md5_password_warnings 默认开启，建议尽快迁移到 SCRAM。
低影响
1

评估 & 备份
