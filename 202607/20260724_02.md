# 一本把 Claude Code "武装到牙齿"的中文实战手册

> 如果你已经用过 Claude Code，却总觉得它"聪明但不持久" —— 看完这本 19 章的中文教程，你会重新理解什么叫 **Agent 元执行层**。

 

## 为什么我们需要 Ruflo？

Claude Code、Cursor、Continue.dev 已经很强，但当任务**跨文件、跨 agent、跨会话、跨机器**时，单兵作战的 LLM 会暴露三个致命短板：

- **没有共享内存** —— 每个 session 是一座孤岛
- **没有协同机制** —— 一个进程扛下所有事，token 暴涨
- **没有持续学习** —— 同样的错误犯两遍

**Ruflo** 就是为了补齐这三个短板而生的。它不是新模型，而是给 Claude Code 与 OpenAI Codex 装上的「**神经系统**」。

> 一句话定义： **Agent = Model + Harness**。模型负责「写」，harness 负责给工具、给内存、给循环、给沙箱、给控制。

 

## 五大核心理念

| #   | 理念                      | 一句话理解                                              |
| --- | ------------------------- | ------------------------------------------------------- |
| ①   | **Model + Harness**       | 包装已有 LLM，提供 314+ MCP 工具、60+ 命令、134 个技能  |
| ②   | **Self-Learning Loop**    | 成功模式自动存入 ReasoningBank，下次优先走通过路径      |
| ③   | **Anti-Drift Defaults**   | 出厂默认小团队 6-8 agents + raft 共识，80% 团队无需调参 |
| ④   | **Zero-Trust Federation** | 跨机器 agent 默认零信任，mTLS + ed25519 + 5 级信任梯    |
| ⑤   | **Truth by Witness**      | Ed25519 签名证明安装字节可信，所见即所信                |

最让我惊艳的是第 ② 条 —— **它有记忆**。这是它与 LangGraph、AutoGen、CrewAI 等"无状态 agent 框架"最本质的区别。

 

## 99% 的人应该这样安装

Ruflo 提供 4 种入口，新人最容易栽在"选错路径"上：

```bash
# ✅ 推荐（99% 团队选这个）
npx ruflo@latest init

# 装完跑三件套
ruflo doctor          # 健康检查（26 项）
ruflo doctor --fix    # 自动修复
ruflo verify          # Ed25519 签名校验
```

`init` 的所有副作用严格局限在 `.claude/`、`.claude-flow/`、`CLAUDE.md` 三个隐藏目录，**绝不碰你的业务代码**。

 

## 它是怎么"自动变聪明"的？

装完第一次对话，**17 个 Hook 自动接管生命周期事件** —— 你保存、提交、回滚的每一刻，hooks 都在后台调用 memory、swarm、routing 逻辑。同时 12 种 worker 默默驻留：audit、optimize、benchmark……无需人工触发就能积累经验。

最终效果： **从"搜命令"变成"说意图"** 。你不再需要记住 314 个工具的名字，只需用自然语言描述目标。
