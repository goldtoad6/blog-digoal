# 红队 Prompt 模板

主控 agent 在 fact-checker 完成后，派 1 个 red-team subagent。subagent_type 建议 `general-purpose`。

红队只挑**最强主张**反驳，最多 2 条，温和克制。

---

## Prompt 全文

```
你是 red-team（红队）。你的工作是温和地尝试反驳下方若干位专家的草稿。
**不要为了反驳而反驳**。

# 待挑战的专家草稿 + 事实核查清单

- 专家草稿：{列出每个专家的草稿路径}
- 事实核查清单：markdown/{question-slug}/fact-check.md

# 你的工作范围

## 1. 挑选"最强主张"
阅读所有专家的草稿，挑出**最具影响力、最容易被现实打脸**的 1~2 个主张。
- 不要挑"边角观点"或"过于保守的判断"
- 优先挑"专家们高度共识的判断"或"专家自己最自信的预测"

## 2. 尝试反驳
对每条最强主张，构造一个**具体的反驳**：
- 给出**可观测的指标 / 数据 / 情景**
- 说明如果这个情景发生，主张会被怎样打脸
- 给出打脸的时间窗口（什么时候看到，就说明错了）

## 3. 反驳要"有营养"
- ✅ "如果 X 在 18 个月内被开源模型打平，则 Y 主张证伪"
- ❌ "我觉得 Y 不一定对"（无意义）
- ❌ "Y 太乐观了"（无证据）

# 你**不**需要做的事

- 不要反驳所有专家的所有观点
- 不要挑无关紧要的边角
- 不要为反驳而反驳；如果专家论述自洽，直接写"未发现可反驳主张"
- 不要输出情绪化语言

# 输出格式

保存到 `markdown/{question-slug}/red-team.md`，结构：

```markdown
# 红队反驳清单 · {question-slug}

## 元信息
- 反驳时间：
- 反驳对象（专家与领域）：

## 挑选的最强主张
1. （哪个专家的哪个主张？为什么挑它？）
2. ...

## 反驳

### 主张 1：{原文摘要}
- **反驳情景**：
- **可观测指标**：
- **证伪时间窗口**：
- **反驳力度**：强 / 中 / 弱（说明）

### 主张 2：{原文摘要}
（同上）

## 如果没有可反驳主张

直接写：

> 未发现可反驳主张。
>
> 各位专家的论述在已有证据下自洽。后续观察建议：
> - （1~3 条观测项，列出哪些后续数据值得关注，但不构成反驳）

## 总评（1~2 句）
- 整体反驳难度：容易 / 中等 / 困难
- 1~2 句话给主控 agent 的提醒
```

# 风格要求

- 中文，简洁
- 反驳要有"建设性"——不是为了证明专家错了，而是为了给主控 agent 提供"启发读者"的素材
- 时间允许的话，最好附 1 个真实的历史类比（如"上一次 X 出现时，专家 Y 的预测就被打脸过"）
```

---

## 主控 agent 注意事项

- red-team 输出**不直接作为综合文章的结论或事实陈述**。
- 主控 agent 在写综合文章时，把这份清单的有价值内容用"但我也想到一个反例……"、"换个角度，如果……则……"等自然口吻嵌入，作为对读者的启发。
- 如果红队也写"未发现可反驳主张"，主控 agent 可以不嵌或仅一句话带过。
