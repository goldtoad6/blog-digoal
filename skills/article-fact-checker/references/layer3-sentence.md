# L3 句子/证据层审查清单

句子层只解决:**每条具体证据、数据、引文、事件是否可信?**

## 6 项必查动作

### ① 反向搜证

把文章中的数据/引文/事件摘出来,**放到搜索引擎找原始来源**,而不是只读文章本身的引用。

```
触发句 → 拆出关键词 → 搜索原始来源 → 比对一致性
```

### ② 查原始出处

看到"XX 研究表明" → 必须找到:
- 论文 DOI / 期刊号 / 卷期页码
- 作者全名 + 机构
- 发表年份
- 论文结论的**原文表述**(不是文章的转述)

**常见手法**:引用真实研究但歪曲结论。

### ③ 核对上下文

引文是否被断章取义?数据是否被剥离语境?

例子:
- 全文结论是"X 在 Y 条件下无效" → 文章只引"X 在 Z 上有显著效果"
- "温度升 0.5℃" 实际是某地某季节均值 → 文章写成"全球升温 0.5℃"
- 实验室小鼠结果 → 直接套用到人体

### ④ 看时间戳

数据/事件/政策的发布日期是否在文章结论的有效期内?

- ☐ 时效内
- ☐ 略过期(可接受,需标注)
- ☐ 严重过期(红旗 — 用旧数据支持新结论)

### ⑤ 看分母

所有百分比、比例、概率数据,必须问:**分母是什么?样本量多少?**

红旗模式:
- "成功率 90%" — 分母=几次?
- "80% 用户喜欢" — 用户基数?抽样方式?
- "专家共识" — 几个专家?

### ⑥ 查利益链

引用来源的资助方、所属机构、过往立场:

- ☐ 无明显利益相关
- ☐ 潜在利益相关(标注)
- ☐ 明确利益相关(红旗 — 降一档可信度)

## 模糊归因红榜(看到这些词要警觉)

| 词 | 问题 | 处理 |
|---|---|---|
| 研究表明 | 哪项研究? | 必查 |
| 专家表示 | 哪位专家? | 必查 |
| 众所周知 | 谁知的? | 红旗 |
| 几乎所有 / 绝大多数 | 多少? | 红旗 |
| 据传 / 据报道 | 转了几手? | 必查 |
| 有人指出 | 谁? | 红旗 |
| 数据表明 | 哪份数据? | 必查 |
| 唯一 / 绝对 / 彻底 | 排除性断言常是伪精确 | 红旗 |
| 整洁数字(73.8%、91.2%) | 真实数据通常更乱 | 可疑 |
| 趋势 / 越来越 | 方向 vs 幅度混用 | 必查 |

## 数字精度陷阱

### 看似精确实则虚假的数字

- 数据精度与样本量不匹配(100 人样本声称精度 0.1%)
- 多个数字间精度不一致(有的给到 4 位,有的粗略)
- 关键数据无误差范围
- 平均数无标准差

### 选择性引用

- 只挑支持结论的数据,不展示分布
- 截取时间窗口(用某月最高点代替全年)
- 选样本(选最支持结论的子集)

## 句子级红旗清单

- 强情感形容词("惊人"、"令人震惊"、"难以置信")
- 模糊量词("许多"、"一些"、"大量")
- 没有主语的被动句("研究表明"、"数据显示")
- 时间副词缺失(过去/现在/将来模糊)
- 反问句代替论证("难道不是吗?")
