# 证据可考性 / 捏造识别 / 源头可信度评估

## 证据是否捏造的检查方法

### 7 步识别法

1. **找原始来源** — DOI / URL / 期刊 / 出版社
2. **核对具体细节** — 日期 / 页码 / 作者 / 机构 / 数据
3. **看图表数据是否自洽** — 图表数字与正文是否一致
4. **反向搜图/数据** — Google 反向搜图 / Scholar 搜数据
5. **询问原作者/机构** — 直接联系或查机构公开档案
6. **检查期刊/出版社是否真实存在** — 警惕"野鸡期刊" / "掠夺性出版"
7. **看其他研究者是否引用** — 高被引 = 真实存在;零引用 = 警惕

### 常见捏造类型

- **完全捏造**:数据、研究、人物、事件全是假的
- **张冠李戴**:真实数据 + 错误归属(把 A 研究说成 B 的)
- **断章取义**:真实研究 + 错误结论(歪曲原意)
- **拼接捏造**:把多个真实事实拼成不存在的关联
- **时效捏造**:旧事件标新日期 / 新事件用旧日期
- **数字精度造假**:伪造高精度数据(如"73.83%")

### 野鸡期刊 / 掠夺性出版识别

- 期刊网站简陋、错字多
- 编委名单查不到或造假
- 审稿周期异常短(< 2 周)
- 收费即发表(无实质审稿)
- 没有清晰的所有者和联系方式
- Beall 列表 / Cabells 黑名单可查

## 证据源头可信度评估

### 来源分层

| 层级 | 类型 | 可信度 |
|---|---|---|
| L1 | 系统综述/meta-analysis / Cochrane / Campbell | 最高 |
| L2 | 同行评议期刊单篇研究 | 高 |
| L3 | 学术专著 / 会议论文 | 较高 |
| L4 | 权威机构报告(政府、WHO、IMF 等) | 高(看立场) |
| L5 | 主流严肃媒体一手报道 | 中高 |
| L6 | 主流媒体评论 / 转引 | 中 |
| L7 | 行业自媒体 | 中低 |
| L8 | 个人博客 / 社交媒体 | 低 |
| L9 | 匿名 / 不可考来源 | 极低 |

### 8 维源头评估

| 维度 | 评估问题 |
|---|---|
| 一手性 | 是一手观察/数据,还是二手转引? |
| 出处可考 | 具体 DOI / URL / 书名 / 章节 / 页码? |
| 作者资质 | 作者在该领域有无专业背景? |
| 利益相关 | 作者/机构有无利益相关方? |
| 同行评议 | 是否经过独立评议? |
| 时效性 | 发表/事件日期?仍在有效期? |
| 可复现性 | 数据 / 方法是否可复现? |
| 多源交叉 | 是否有 ≥2 个独立来源印证? |

### 利益链识别

- 作者所属机构 / 资助方 / 顾问方
- 引用来源的资助方
- 数据提供方
- 文章发布平台的商业关系

**未声明 ≠ 无利益。** 默认按"低可信度"处理。

## 反向搜证操作清单

### 数据反向搜证

```
搜索模板: "<具体数字>" "<单位>" "<年份>"
举例: "73.8%" "糖尿病" "2023"
目的: 找到原始数据集 / 调查方法
```

### 引文反向搜证

```
搜索模板: "<作者>" "<期刊>" "<年份>" "<关键词>"
举例: "Smith J" "Nature" "2022" "microbiome"
目的: 找到原论文,核对作者真实结论
```

### 事件反向搜证

```
搜索模板: "<主体>" "<日期>" "<事件关键词>"
举例: "OpenAI" "2024-03-15" "董事会"
目的: 找到事件的一手报道或官方声明
```

### 概念反向搜证

```
搜索模板: "<理论名>" site:.edu OR site:.gov OR site:.org
举例: "比较优势理论" site:.edu
目的: 找到权威定义 / 学术解释
```

## 反向搜证红旗

| 信号 | 含义 |
|---|---|
| 搜不到任何原始来源 | 可能是捏造 |
| 找到原始但结论被歪曲 | 断章取义 |
| 找到原始但日期不对 | 时效捏造 |
| 找到原始但作者身份不符 | 张冠李戴 |
| 多源数据但相互引用 | 自循环引用,价值低 |
| 仅社交媒体流传 | 源头不可考 |

## 输出格式

每条证据的可考性结果填入 L3 评分表,问题严重的进问题清单并标严重度。
