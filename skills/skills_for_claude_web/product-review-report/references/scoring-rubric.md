# 评分细则参考手册

各维度评分标准（1~5 分制）。评分时应综合考量，有明确数据支撑的按数据打分，无数据时根据用户反馈和公开信息主观评估，并在报告中注明"基于公开资料推断"。

---

## 通用评分原则

| 分值 | 含义 | 参考描述 |
|------|------|----------|
| 5 | 行业标杆 | 超出同类产品，有明确数据证明的领先优势 |
| 4 | 表现良好 | 达到或略超行业平均水平，用户满意度高 |
| 3 | 基本合格 | 满足基本需求，但有明显可改进空间 |
| 2 | 存在缺陷 | 多个已知问题影响正常使用，或用户投诉较多 |
| 1 | 不可接受 | 核心功能缺失或严重 bug 未修复，不建议使用 |

## 决策者维度评分细则

### 总拥有成本（TCO）
- **5 分**：订阅/授权费透明，无隐性费用，实施成本低（<2 周上线），年维护费率 <15%
- **4 分**：定价清晰，偶有附加模块费用，实施成本合理（2~6 周），年维护费率 15~20%
- **3 分**：定价复杂（按用量/席位/模块叠加），实施周期 1~3 个月，维护费率 20~25%
- **2 分**：定价不透明，需复杂谈判，实施周期 3~6 个月，存在意外超支风险
- **1 分**：定价极不透明，强制捆绑服务，历史上有客户投诉超支的公开记录

### ROI 预期
- **5 分**：有同类案例证明的明确 ROI 数据（如"节省 40% 人力"），且有第三方报告背书
- **4 分**：厂商提供 ROI 案例，内容具体但未经第三方验证；或行业共识 ROI 较高
- **3 分**：ROI 可预期但难以量化，依赖具体实施质量
- **2 分**：ROI 主要来自厂商宣传，缺乏实际案例支撑
- **1 分**：无可信 ROI 案例，甚至有客户反映负 ROI

### 厂商可靠性
- **5 分**：上市公司或知名基金 B+ 轮以上融资，存续 5 年以上，500+ 企业客户，明确 SLA（99.9%+）
- **4 分**：A 轮以上融资或盈利运营，存续 3~5 年，100~500 客户，有 SLA
- **3 分**：种子/天使或早期 A 轮，存续 1~3 年，数十家客户，有承诺但无详细 SLA
- **2 分**：融资情况不透明，存续 <1 年，客户数极少，无 SLA 承诺
- **1 分**：有倒闭风险迹象、核心团队离职、融资断裂的公开报道

### 合规与安全
- **5 分**：通过 SOC 2 Type II + ISO 27001 + 行业特定认证（如 HIPAA/等保三级），数据合规文档完善
- **4 分**：通过 SOC 2 Type I 或 ISO 27001，等保二级，有数据处理协议（DPA）
- **3 分**：基础安全措施齐全（HTTPS、加密存储），但缺乏第三方认证
- **2 分**：安全措施薄弱，文档不完整，或有历史数据泄露记录
- **1 分**：有严重安全漏洞公开记录或监管处罚记录

### 战略契合度
- **5 分**：原生支持主流技术栈，开放 API（REST/GraphQL），无供应商锁定，明确的 3 年路线图
- **4 分**：良好 API 支持，迁移路径清晰，路线图公开但更新频率一般
- **3 分**：基础集成能力，部分数据可导出，路线图需签 NDA 才可查看
- **2 分**：集成能力有限，数据迁出困难，路线图不透明
- **1 分**：封闭系统，数据锁定，无公开路线图，集成依赖高价定制开发

## 使用者维度评分细则

### 易用性
- **5 分**：无需培训即可上手（<1 天），界面直觉式，操作路径短（关键任务 <3 步）
- **4 分**：<3 天上手，界面清晰，偶有不直觉的操作
- **3 分**：1~2 周熟练，有完整培训材料，学习曲线适中
- **2 分**：需要 >2 周专项培训，UI 陈旧，操作步骤繁琐
- **1 分**：界面反直觉，严重依赖技术背景，新用户错误率高

### 性能与稳定性
- **5 分**：核心操作响应 <200ms，过去 12 个月可用性 >99.9%，无严重故障公开记录
- **4 分**：响应 <500ms，可用性 99.5~99.9%，偶有轻微故障但快速恢复
- **3 分**：响应 <2s，可用性 99~99.5%，有过故障但均在 SLA 内恢复
- **2 分**：高峰期响应 >3s 或有明显卡顿，可用性 <99%，有影响业务的故障记录
- **1 分**：频繁宕机或响应超时，严重影响日常使用

### 核心功能完整度
根据用户指定的功能点逐项评分，综合取平均后按以下标准定级：
- **5 分**：所有功能点全部覆盖且超出预期，有同类最佳的独特功能
- **4 分**：核心功能点全覆盖，少量边缘需求需要 workaround
- **3 分**：80% 核心需求覆盖，有 1~2 个重要功能缺失但有计划
- **2 分**：仅覆盖 60% 核心需求，关键功能缺失且无路线图
- **1 分**：核心功能严重缺失，与宣传不符

### 集成与扩展性
- **5 分**：完善 REST/GraphQL API，SDK 覆盖主流语言，300+ 原生集成，Webhook 支持
- **4 分**：完整 API，50~300 原生集成，良好文档
- **3 分**：基础 API，10~50 集成，或依赖 Zapier/Make 等中间件
- **2 分**：API 不完整或文档差，集成需大量定制开发
- **1 分**：无 API 或 API 极其有限，几乎不可集成

### 文档与技术支持
- **5 分**：文档完整且实时更新，有交互式 Demo/Sandbox，社区活跃，支持响应 <4h（工作日）
- **4 分**：文档较完整，有官方论坛，支持响应 <24h
- **3 分**：基础文档，支持响应 1~3 天，社区较小但有效
- **2 分**：文档过时或不完整，支持响应 >3 天，社区几乎无活跃
- **1 分**：几乎无文档，支持极难触达或无响应
