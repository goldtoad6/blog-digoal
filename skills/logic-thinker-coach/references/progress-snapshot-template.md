# 进度摘要模板

每完成一轮（每答完一题）或用户主动要求时，输出**进度摘要**。这份摘要是跨会话延续训练的桥梁。

**摘要必须用户可粘贴、可读懂**——下次开局用户把摘要贴在第一条消息里，AI 就能立刻恢复训练上下文，不需要重新诊断。

---

## 摘要触发时机

1. **诊断收尾** → 输出 v1
2. **每 5 轮训练后** → 自动输出 v2、v3……
3. **每 10 轮训练后** → 升级为"阶段复盘"格式（见 SKILL.md 模式三）
4. **用户主动要求**（"出进度"、"总结一下"、"我学到啥了"）→ 立即输出
5. **每轮结尾** → **不输出完整摘要**，但心里更新状态；下次第 5 轮时输出

---

## 摘要标准格式（v1 / v2 / v3…）

```markdown
# 进度快照 · v{N}

**日期**：YYYY-MM-DD
**总训练轮次**：{N}（不计诊断）
**当前档位**：<1 / 2 / 3 / 4>

## 已覆盖维度
- 维度 X（出现 N 次，平均分 M/5）
- 维度 Y（出现 N 次，平均分 M/5）
- ...

## 强项
- {具体能力 1}（例：能识别 trade-off 的真实代价）
- {具体能力 2}

## 待改进
- {具体问题 1}（例：容易在反事实推演时把"现在的结果"当成"必然"）
- {具体问题 2}

## 反复出现的模式
- {模式描述 1}（例：连续 3 题把"动机"当"原因"）
- {模式描述 2}

## 训练重点（接下来的 5-10 题优先练）
- 维度 X：因为 {为什么优先}
- 维度 Y：因为 {为什么优先}

## 上一轮要点（仅 v2 及以后）
**题目**：{一句话}
**用户回答要点**：{2-3 句话概括，不复述全文}
**本轮最关键的发现**：{一句具体反思}

## 下次开局请粘贴本快照
```

## 维度得分追踪（内部维护）

每个维度的得分在心里维护：

| 维度 | 出现次数 | 最近 3 次得分 | 平均分 | 趋势（↑→↓） |
|------|---------|-------------|--------|------------|
| 定义与边界 | | | | |
| 第一性原理 | | | | |
| 因果链与副作用 | | | | |
| 假设检验 | | | | |
| 反事实与逆推 | | | | |
| 权衡与取舍 | | | | |
| 元认知 | | | | |
| 框架迁移 | | | | |
| 立场反转 | | | | |
| 不确定性管理 | | | | |

**这个表本身不出现在摘要里**，但它是判断"训练重点"的依据：

- 维度出现 0 次 → 优先补一题
- 维度出现 ≥ 3 次但平均分 ≤ 2 → 反复练
- 维度出现 ≥ 5 次且平均分 ≥ 4 → 可以降频

## 阶段复盘格式（每 10 轮）

```markdown
# 阶段复盘 · 第 {N} 轮训练

**训练跨度**：YYYY-MM-DD → YYYY-MM-DD
**总轮次**：{N}
**档位变化**：v1 档位 → 当前档位

## 一、整体轨迹
- 累计覆盖了 {M} 个维度（满分 10）
- 各维度平均分：{列表}
- 整体档位提升情况

## 二、反复出现的模式
### 优点（≥ 3 次出现）
- {模式 1}
- {模式 2}

### 弱点（≥ 3 次出现）
- {弱点 1}
- {弱点 2}

## 三、盲点地图
### 长期薄弱（出现 ≥ 3 次但仍 ≤ 2 分）
- 维度 X：{具体表现}
- 维度 Y：{具体表现}

### 从未练过
- {列出未覆盖的维度}

## 四、下阶段训练建议（接下来 10 轮）
1. 优先练：{维度}（因为 {理由}）
2. 刻意打破舒适区：{具体策略}
3. 场景多样化：{建议引入的新场景}
4. 是否需要调档：{建议}

## 五、关键转折点（值得记录的精彩瞬间）
- 第 X 轮：{一句话描述}

## 六、下一阶段目标
- {一句话具体可衡量的目标，例："反事实推演能拉到第二层因果"}
```

---

## 简短版（每轮结尾用）

如果用户希望"每轮结尾也输出进度"，可以输出**极简版**：

```
✓ 第 N 轮 | <维度> | <档位 X> | <本轮亮点一句话> | <本轮缺口一句话>
```

不要每轮都贴完整摘要——用户会视觉疲劳。

---

## 摘要的元规则

1. **不要列所有题**——只列用户能从中学习的
2. **不要写"用户表现良好"这种空话**——给具体证据
3. **不要预设用户下一轮要练什么**——给建议而不是指令
4. **长度控制**：
   - v1、v2、v3 等常规摘要：200-400 字
   - 阶段复盘：500-800 字
   - 简短版：单行

5. **让用户知道怎么用**——每份摘要结尾都明确写"下次开局请粘贴本快照"

---

## 摘要的常见错误

❌ **流水账**：把所有题目列一遍，没有抽象
✅ **抽象模式**：从多题中提炼反复出现的模式

❌ **空泛表扬**："用户整体表现稳定，逻辑能力较好"
✅ **具体观察**："用户能在 4/5 题中识别反例，但反例识别后未进一步追问'这个反例是否动摇核心论点'"

❌ **全维度平衡建议**："建议均衡训练所有维度"
✅ **针对性建议**："接下来优先练'框架迁移'——过去 10 题中用户在此维度平均 1.8 分，且从未主动使用过类比/跨领域抽象"

❌ **目标模糊**："希望下一阶段有所提升"
✅ **目标可衡量**："目标：'反事实推演'维度的平均分从 2.0 提升到 3.0"
