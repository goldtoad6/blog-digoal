# 产品评测框架

> 加载时机：完成信息收集后、撰写报告前必读。
> 本文件定义评测的维度、权重、双视角关注点，用于统一输出标准。

## 一、双视角核心区别

### 决策者视角（买/用不用？投不投钱？）
关心的是**结果和风险**，不看过程细节：
- **业务价值**：能否解决业务问题、带来什么收益（效率提升/收入增长/成本降低）
- **ROI 与成本**：许可费/订阅费、部署成本、迁移成本、运维成本、人力成本
- **战略契合度**：是否符合技术路线、组织能力、生态战略
- **风险评估**：供应商锁定风险、厂商稳定性、合规与安全、数据主权
- **替代方案**：与竞品的差异化、与现有系统的可替代性
- **决策可逆性**：选了之后撤换的成本有多高

### 使用者视角（好不好用？值不值得学？）
关心的是**日常体验**，结果要好用才行：
- **上手成本**：安装/注册/配置难度、学习曲线陡峭度
- **日常效率**：完成核心任务需要几步、是否有快捷路径
- **稳定性与性能**：卡顿、崩溃、响应延迟、并发能力
- **可定制性**：能否调整符合个人工作流、插件/扩展丰富度
- **体验细节**：UI 合理性、错误提示、文档质量、社区活跃度
- **故障恢复**：出问题找谁、文档/社区能否自救

## 二、评测维度模板

按"产品类型"选用对应维度组合（不全用，匹配即可）：

| 维度族 | 适用产品类型 | 关键指标 |
|--------|------------|---------|
| 功能完备性 | 通用 | 核心功能、边界情况、覆盖率 |
| 性能 | 计算/存储密集 | 吞吐、延迟、资源占用、扩展性 |
| 易用性 | 通用 | 上手时间、UX 评分、文档质量 |
| 生态与集成 | 通用 | API/SDK、第三方集成、插件市场 |
| 安全与合规 | ToB/数据敏感 | 认证授权、加密、审计、GDPR/等保 |
| 成本 | 通用 | License、运维、迁移、培训、机会成本 |
| 厂商与社区 | 通用 | 厂商背景、融资、Roadmap、社区活跃度 |
| 可扩展性 | 平台型 | 二次开发、定制、API 限制 |
| 可靠性 | 关键业务 | SLA、容灾、备份、故障率 |
| AI/智能化 | AI 类 | 模型能力、推理成本、幻觉率、私有化能力 |

## 三、竞品对比维度

每个对比项打分建议使用 1-5 颗星：
- ⭐ 极差 / 不支持
- ⭐⭐ 勉强可用
- ⭐⭐⭐ 合格
- ⭐⭐⭐⭐ 优秀
- ⭐⭐⭐⭐⭐ 行业领先

对比表中**每行**都应能从产品文档、官方说明、第三方测评、用户反馈中找到依据。**禁止编造数据**。

## 四、结论输出规范

### 决策者结论模板
```
**建议：[采购/试用/观望/放弃]**
- 核心优势（1-2 句）
- 核心风险（1-2 句）
- 关键决策点：[是/否需要 X 功能、是否可接受 Y 锁定、预算是否匹配]
- 推荐人群：[适用规模、行业、团队画像]
- 不推荐人群：[不适用的场景]
```

### 使用者结论模板
```
**上手难度：[低/中/高]**
- 核心痛点解决：[能/不能/部分解决]
- 学习曲线：[描述 1 句话]
- 替代品切换成本：[低/中/高]
- 推荐的上手路径：[3-5 步]
```

## 五、图示选型指引

- **架构图 / 流程图 / 决策树 / 时序图 / 状态图 / 类图**：用 Mermaid
- **组织结构 / 业务关系**：用 Mermaid
- **复杂数据可视化 / 自定义图形 / Logo/品牌色块**：用 SVG（必须外挂）
- **简单结构 / 文字密集场景**：用 ASCII Text 图

**SVG 外挂规则**：
1. SVG 文件保存到与 markdown 同目录的 `svg/` 子目录或同级目录
2. 文件命名规范：`{report-name}-{chart-name}.svg`（小写连字符）
3. markdown 中用 `![描述](./svg/xxx.svg)` 引用
4. SVG 内禁止外链资源（防止链接失效），字体用系统通用字体
